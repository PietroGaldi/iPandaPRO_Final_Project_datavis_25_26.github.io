Methodology of the project

This section outlines the data sources, processing pipelines, and design decisions behind the visualization of the RAISE ecosystem.

Data sources

The data used to generate the visualizations were retrieved from OpenAlex, an open catalog of works, authors, institutions, and related metadata.

Relevant publications were identified by querying funding information associated with RAISE (code ECS00000035). Unique identifiers linked to these funded outputs were then used to retrieve the corresponding records from OpenAlex.

This process ensured a consistent and structured dataset covering publications, authorship, institutional affiliations, venues, and geographic information. All subsequent data processing, aggregation, and transformation steps were performed on the OpenAlex metadata to support the analytical and visual components of the project.

Data processing & cleaning

The primary dataset is stored in openalex_works_full.csv, which aggregates all information retrieved via the OpenAlex API, including a raw JSON column containing the original data.

To optimize data handling, Python scripts were employed to generate two specific CSV files: openalex_people.csv, which details the researchers involved, and institutions_osm_coords.csv, which lists the participating institutions. Geographic coordinates were geocoded into the institutions file using OpenStreetMap, with manual corrections applied to ensure locational accuracy.

People page

  • Entities: Since the raw data separates authors and works, a link was created between each researcher and their publications. This allows the calculation of metrics such as "Total Works per Researcher" and "Primary Research Topic".
  • Topic analysis: The raw_json column contains categorization levels: Level 0 (Domain) and Level 1 (Topic). Authors are classified based on the dominance of these concepts in their publication history.

Institutions page

  • Choropleth map: Geospatial data was integrated with publication records by extracting and normalizing country codes. The pipeline aggregated the count of distinct institutions per country.
  • Treemap: The pipeline processed affiliation data to calculate publication volume per institution. The visualization uses a squarified layout where area corresponds to publication count.

Collaborations page

  • People Network: Unique author identifiers were mapped to their institutions. Relationships were quantified by iterating through publications to identify co-authorship pairs.
  • Flow map: Nodes were positioned according to geographic coordinates. Links represent unique pairs of institutions listed on the same publication, weighted by collaboration frequency.

Limitations & Constraints

The visualizations presented are subject to data quality limitations inherent to the source, OpenAlex. Because OpenAlex relies on automated algorithms to aggregate vast amounts of bibliographic data, it lacks human manual curation.

Consequently, the dataset contains noise; for example, the use of author aliases may result in disambiguation errors, where a single physical researcher is represented by multiple distinct author profiles. This limitation extends to the classification of categories and topics, which are ingested directly from OpenAlex and thus subject to the same automated tagging inaccuracies.

Additionally, the project is subject to temporal constraints. As the RAISE project is currently ongoing, the data represents a snapshot in time and is necessarily incomplete regarding the final output of the research.
